# Предложение по очистке проекта

Ниже — безопасный, предложенный список изменений для приведения репозитория в более аккуратный и профессиональный вид. Ни одно действие не выполнено — это предложение. После вашего одобрения я могу выполнить эти операции автоматически через `scripts/cleanup.ps1`.

## Рекомендованные действия (безопасные/обратимые):

- Переместить архивы и бизнес-планы из `attached_assets/` в `docs/attached_assets/` — чтобы держать исходники документации отдельно от кода.
- Добавить README в `client/public/assets/` и добавить `.gitkeep` при необходимости.
- Оставить в корне только нужные markdown-файлы (`README.md`, `TECHNICAL_DOCUMENTATION.md`, `SECURITY.md`) — остальные планы/черновики переместить в `docs/`.

## Предложенные удаления (после проверки):

- Временные или тестовые файлы в `client/public/` (например, `test-challenge.json`) — удалить, если используются только для отладки.
- Удалить автогенерируемые/временные файлы, если таковые есть (например, `dist/` и другие сборки) — только если вы уверены что их можно восстановить из git или сборки.

## Как это будет работать

1. Я подготовил скрипт PowerShell `scripts/cleanup.ps1`, который сначала делает dry-run (показывает, что будет удалено/перемещено).
2. После вашего подтверждения можно запустить скрипт с флагом `-Run` и он выполнит операции.
3. Всякий удалённый/перемещённый материал можно восстановить из git (если изменения закоммичены) или из резервной копии — скрипт предложит создать резервную папку перед удалением.
