# Workflow и правила для команды CyberWhale

1. Создание задачи
- Описывайте задачу в формате Markdown по шаблону `tasks/000-template.md`.
- Нумерация локальных задач: `NNN` (скрипт `scripts/new_task.ps1` создаёт следующий `NNN`).
- Код в диаграмме Ганта — ваш внешний код (например, `43101`).

2. Статусы задач
- `open` — новая задача
- `in-progress` — в работе
- `review` — ожидает проверки
- `done` — завершена

3. Как добавить задачу
- Запустить PowerShell: `.\	asks\new_task.ps1 -code 43123 -title "Feature name" -assignee "@dev" -start 2025-11-24 -end 2025-11-30`
- Скрипт обновит `tasks/index.json` и создаст файл с номером.

4. Gantt Viewer
- Откройте `docs/gantt.html` при запущенном `npm run dev` (server должны отдавать /tasks/index.json из корня).
- Клик по строке открывает файл задачи в новой вкладке.

5. Отчётность
- Используем `tasks/index.json` как единую «источник правды» о сроках и прогрессе.
- По завершении спринта экспортируем `tasks/index.json` и формируем отчет.
